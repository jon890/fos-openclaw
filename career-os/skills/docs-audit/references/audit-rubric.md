# docs-audit 분류 및 품질 평가 기준

## 분류 상태

### keep
- 현재 구조/운영과 여전히 맞는다
- 다른 문서가 대체하지 않는다
- 다시 읽을 가치가 높다
- 역할이 비교적 분명하다

### refresh-needed
- 핵심은 여전히 유효하다
- 최신 구조/정책/경로와 어긋난다
- 링크, 예시, 문장, 결론 일부 업데이트가 필요하다

### merge
- 다른 문서와 내용이 상당히 겹친다
- 둘을 유지하는 비용이 더 크다
- 하나의 상위 문서 또는 허브 문서로 합치는 편이 낫다

### archive
- 당시 맥락에서는 의미 있었지만, 현재 운영 기준으로는 직접 효용이 낮다
- 삭제할 정도는 아니지만 기본 탐색 경로에서는 비활성화하는 편이 낫다

### delete-candidate
- 중복이 심하고
- 최신 문서가 사실상 대체하고 있으며
- 유지 비용 대비 의미가 거의 없다
- 실제 삭제 전에는 사람 확인이 필요하다

## 품질 평가 축

### 1. clarity
- 문서가 한 문장으로 무엇을 설명하는지 말할 수 있는가?
- 메타 서술이 많고 핵심이 흐리지는 않는가?

### 2. overlap
- 같은 설명을 다른 문서와 길게 반복하는가?
- 중복 대신 링크로 넘길 수 있는가?

### 3. role-fit
- 이 문서는 허브인가, 심화 문서인가, 사례 문서인가, 레퍼런스인가?
- 그 역할이 문서 내용과 실제로 맞는가?

### 4. link-quality
- 관련 문서로 자연스럽게 이동할 수 있는가?
- 허브 -> 심화 -> 사례 흐름이 보이는가?

### 5. practical-value
- 공부나 실무에 바로 도움이 되는가?
- 예시, 실수 패턴, 선택 기준이 있는가?

### 6. interview-value
- 면접 대비 문서라면 답변 구조를 만드는 데 도움이 되는가?
- 단순 개념 나열에서 멈추지 않는가?

### 7. maintenance-cost
- 이 문서를 계속 유지할 가치가 있는가?
- 너무 좁은 임시 메모가 영구 문서처럼 남아 있지는 않은가?

## 감사 시 질문
- 이 문서는 지금도 미래 작업에 도움 되는가?
- 최신 구조와 충돌하지 않는가?
- 더 좋은 최신 문서가 이미 있는가?
- 문서 역할이 분명한가?
- 중복 설명을 줄이고 링크 구조로 바꾸는 편이 낫지 않은가?
- 최근 diff가 실제로 품질을 높였는가, 아니면 단지 문장을 옮겼을 뿐인가?
