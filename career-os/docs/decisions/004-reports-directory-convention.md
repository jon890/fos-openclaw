# ADR-004: reports/ 디렉터리 컨벤션 [폐기]

## 상태

**폐기 (2026-04-18).** 최상위 `reports/` 큐레이션 공간은 ADR 작성 후 실제로 한 번도 사용되지 않았다.
사용되지 않는 관례를 유지하는 것이 오히려 "저 폴더가 뭐지?"라는 혼란을 만들었고, `data/reports/` 한 곳으로 규약을 단순화하기로 했다.

- `career-os/reports/` 디렉터리는 삭제됨
- 자동 생성 리포트는 계속 `data/reports/`에만 기록
- 큐레이션이 필요한 경우 `data/reports/` 안에서 직접 인용하거나 fos-study에 커밋

이하 내용은 이력 보존용.

---

## 배경

프로젝트에 두 개의 reports 경로가 공존했다.

- 최상위 `reports/` — 비어있었으나 생성되어 있음
- `data/reports/` — 스크립트가 실제로 읽고 쓰는 경로

의도가 불명확해 "리포트가 어디 있지?"라는 혼란이 생길 수 있는 구조였다.

## 결정 (당시)

두 디렉터리의 역할을 명확히 분리한다.

| 경로 | 역할 | 작성 주체 |
|------|------|-----------|
| `data/reports/` | 자동 생성 리포트 (baseline, daily) | 스크립트 (`run_baseline.sh`, `run_daily.sh`) |
| `reports/` | 정제된 참조 리포트 (사람이 보관할 핵심본) | 수동 (사람이 직접 복사/편집) |

자동 생성본 중 특히 유용한 리포트를 보존하거나 편집하고 싶을 때 `reports/`에 저장한다.
예: 최종 면접 전날 정리한 핵심 약점 요약본.

## 결과 (당시)

- 스크립트는 항상 `data/reports/`에만 쓴다 (변경 없음)
- `reports/`는 사람의 큐레이션 공간으로 명확히 정의
- `AGENTS.md`에 이 컨벤션을 기술해 혼란 방지
